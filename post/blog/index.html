Github&Gridea&Vercel

0.折腾

  • Hexo ：快速、简洁且高效的博客框架；
  • Hugo：全球最快的网站建设框架；
  • Gridea ：一个静态博客写作客户端。

一直在这三个框架之间不停的切换，换来换去，却发现折腾的时间远远的超过了写Blog的时间。在历经 Hexo 和 Hugo 的磨难之后，还是切换回了Gridea 。

1.Github

1.1.GitHub Page

新建 GitHub 账号后新建一个公开仓库，名为ikulkix.github.io。

1.2.Token

点击这里创建一个 Token，勾选上 repo 的相关权限即可。生成之后记得把 Token 复制到你的本地，因为一旦关闭网页将不能再看到它。

2.Gridea

2.1.Gridea

Github
Gridea 主页
示例网站

2.2.评论系统

Gitalk
DisqusJS 评论系统

2.3.配置

根据官方指南快速上手进行配置，这里有个问题，Github主分支现在为 main 个人博客建议使用 main 做远程分支。

Gridea同步后的Repository

3.Vercel

vercel在台湾，GitHub pages在美国旧金山，Gridea 部署到 GitHub pages 并启用 vercel 加速&域名

3.1.登陆

直接使用你的Github账号关联登陆。

3.2.Connected Git Repository

点击这里导入Repository。

3.3.域名

仓库导入后，便可在项目设置中绑定域名或者使用Vercel提供的二级域名，这里绑定为www.ikulkix.com。

3.4.结束

按照上面做完以后，就能得到一个地址为https://www.ikulkix.com的博客。
